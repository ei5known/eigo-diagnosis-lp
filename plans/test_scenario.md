## E2E手動テストシナリオ

**注記:** このテストには、実際にメールを受信できる本物のメールアドレスを使用してください。

### 1. [STEP1: LP診断] LPへのアクセスから、診断フォームの回答と送信。
- [ ] ブラウザを開き、LP [`index.html`](index.html) にアクセスする。
- [ ] 診断フォームの各項目にテストデータを入力する。
- [ ] フォームを送信するボタンをクリックする。
- [ ] フォーム送信後、LP診断完了画面に遷移したことを確認する。

### 2. [自動メール①の確認] 診断サンクスメールの受信確認と、メール内リンクからの予約フォームへの遷移。
- [ ] 入力したメールアドレスの受信トレイを確認し、「LP診断サンクスメール」という件名のメールが届いていることを確認する。
- [ ] メール本文中の「予約フォームへ進む」などのリンクをクリックし、予約フォーム ([`booking.html`](booking.html)) に正しく遷移することを確認する。

### 3. [STEP1: 予約フォーム] 予約フォームでの氏名・メール・事前カルテ等の入力、送信、およびカレンダー画面への遷移確認。
- [ ] 予約フォームの氏名、メールアドレス、事前カルテなどの必須項目にテストデータを入力する。
- [ ] フォームを送信するボタンをクリックする。
- [ ] フォーム送信後、カレンダー選択画面または予約完了画面に正しく遷移したことを確認する。

### 4. [自動メール②の確認] 予約確定メールの受信確認と、メール内リンク（URLパラメータ付き）からのSTEP2への遷移。
- [ ] 入力したメールアドレスの受信トレイを確認し、「予約確定メール」という件名のメールが届いていることを確認する。
- [ ] メール本文中の「面談前アンケートへ進む」などのリンク（URLパラメータ付き）をクリックし、STEP2 ([`step2.html`](step2.html)) に正しく遷移することを確認する。

### 5. [STEP2: 面談前アンケート] 面談前アンケートの回答と送信。
- [ ] STEP2の面談前アンケートの各項目にテストデータを入力する。
- [ ] アンケートを送信するボタンをクリックする。
- [ ] アンケート送信後、STEP2完了画面に遷移したことを確認する。

### 6. [自動メール③の確認] STEP2受信確認メールの受信確認。
- [ ] 入力したメールアドレスの受信トレイを確認し、「STEP2受信確認メール」という件名のメールが届いていることを確認する。

### 7. [データベース確認] 最後に、Googleスプレッドシート（diagnosis_results, karte_results, step2_results）にテストデータが正しく保存されているかの確認手順。
- [ ] Googleアカウントにログインし、関連するGoogleスプレッドシート（`diagnosis_results`, `karte_results`, `step2_results`）を開く。
- [ ] 各スプレッドシートで、今回入力したテストデータが新しい行として正しく追加され、各列に期待通りに保存されていることを確認する。
    - [ ] `diagnosis_results` スプレッドシートには、LP診断フォームで入力したデータが保存されていることを確認。
    - [ ] `karte_results` スプレッドシートには、予約フォームで入力した氏名、メール、事前カルテのデータが保存されていることを確認。
    - [ ] `step2_results` スプレッドシートには、STEP2面談前アンケートで入力したデータが保存されていることを確認。
- [ ] 各スプレッドシートのデータが、テスト実施日時と紐づいているか、または最新のデータとして確認できるかを確認する。
